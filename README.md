# ai-copywriter-v1

这是一个本地运行的 AI 私域文案生成器网页演示版。

当前版本为 Final Pro 版，已支持真实 AI 生成，需要配置 OpenAI API Key。项目使用 Node.js 本地后端调用 OpenAI API，API Key 保存在本地 `.env` 文件中，不会写进前端文件。

## 项目功能

- 页面顶部展示科技感 Hero：首页、产品 Logo、状态标签、版本标签和功能亮点。
- 展示“产品价值”“适合谁用”“使用步骤”“案例展示”等演示模块。
- 输入推广需求。
- 选择行业，已覆盖餐饮美食、美业、本地生活、电商、短视频、自媒体、软件工具、AI 服务、私域社群、知识付费等常见行业。
- 支持自定义行业。
- 选择文案风格。
- 填写目标客户。
- 填写产品卖点。
- 调用 OpenAI API 生成 4 类中文文案：微信朋友圈、抖音短视频、小红书种草、私聊客户成交话术。
- 生成时显示“生成中，请稍等...”，避免重复点击。
- 生成成功后显示成功提示和本次生成耗时。
- 生成失败时显示更清楚的中文原因，例如 API Key、余额、权限、网络或服务异常。
- 新增“7 天朋友圈内容计划”，一键生成 7 天私域朋友圈内容。
- 7 天内容包含朋友圈标题、正文内容、配图建议、适合发布时间和运营目的。
- 支持复制 7 天内容。
- 支持导出 7 天内容为 TXT 文件。
- 每类文案支持一键复制。
- 支持复制全部文案。
- 支持导出为 TXT 文件。
- 支持重新生成。
- 支持示例填充。
- 支持清空内容。
- 底部展示科技感微信咨询 CTA。
- 页面显示当前版本：Final Pro 版。
- 手机和电脑浏览器都可以打开演示。

## 项目文件

- `index.html`：网页结构，包含输入框、下拉选择、按钮、4 类文案展示区域和 7 天朋友圈内容计划模块。
- `style.css`：Final Pro 科技感页面样式，包含深色背景、玻璃拟态卡片、渐变按钮、响应式布局。
- `script.js`：前端交互逻辑，调用本地接口 `POST http://localhost:3000/api/generate` 和 `POST http://localhost:3000/api/generate-weekly-plan`。
- `server.js`：Node.js 本地后端，读取 `.env` 中的 API Key 并调用 OpenAI API，提供单条文案和 7 天计划两个接口。
- `package.json`：项目依赖和启动命令。
- `.env.example`：环境变量示例文件。
- `.gitignore`：忽略 `.env` 和 `node_modules`，避免密钥和依赖目录被提交。
- `README.md`：项目说明和运行方法。

## 安装依赖

进入项目目录：

```bash
cd ~/AI_Workspace/ai-copywriter-v1
```

安装依赖：

```bash
npm install
```

## 创建 .env

复制示例文件：

```bash
cp .env.example .env
```

打开 `.env`，把里面的内容改成你自己的 OpenAI API Key：

```bash
OPENAI_API_KEY=你的真实OpenAI API Key
```

不要把 API Key 发给别人，也不要写进 `index.html`、`style.css`、`script.js`。

## 启动本地服务

```bash
npm start
```

启动成功后，终端会显示：

```bash
AI 私域文案生成器已启动：http://localhost:3000
```

## 打开网页测试

在浏览器打开：

```text
http://localhost:3000
```

测试步骤：

1. 点击“示例填充”。
2. 点击“生成文案”。
3. 查看 4 类文案是否生成。
4. 点击“生成 7 天朋友圈内容”。
5. 查看 7 天内容卡片是否生成。
6. 测试“一键复制”“复制全部文案”“导出 TXT”“复制 7 天内容”“导出 7 天内容 TXT”“重新生成”“清空内容”。

自定义行业测试：

1. 在“行业选择”中选择“自定义行业”。
2. 在出现的输入框里填写自己的行业，例如“茶馆”“留学咨询”“线下沙龙”。
3. 填写需求、目标客户和产品卖点。
4. 分别点击“生成文案”和“生成 7 天朋友圈内容”。
5. 如果没有填写自定义行业，页面会提示“请填写自定义行业。”

如果没有配置 `.env` 或没有填写 `OPENAI_API_KEY`，页面会提示：

```text
请先配置 OpenAI API Key。
```

如果生成失败，可以先检查：

1. `.env` 里是否已经填写正确的 `OPENAI_API_KEY`。
2. 本地服务是否正在运行。
3. OpenAI API 账户是否有余额或模型权限。
4. 终端里的服务日志是否出现“OpenAI generation failed”。日志不会打印 API Key。
